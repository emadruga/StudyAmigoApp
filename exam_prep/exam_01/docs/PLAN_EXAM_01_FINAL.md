## 1. Purpose and Context

### 1.1 Background

This is the **summative final exam for the first bimester** (Prova de Final de 1º Bimestre). It follows the same structural model as the Mid-E01 Prep Exam (`exam_prep/docs/PLAN_PREP_EXAM.md`), adapted for three differences:

1. **Graded assessment** — unlike the formative prep exam, this exam counts for a grade.
2. **Three-tier structure** — Tier 3 (Expanding) students are now in scope; if no Tier 3 students are confirmed, Section 3 can be omitted at generation time.
3. **New question bank** — questions must be distinct from those in `exam_prep/bases/prep_exam_bank.json` to prevent recall advantage from the earlier prep exam.

### 1.2 Subject

**Verbal Tenses** — the same content domain as E01 and the prep exam. Students have been exposed to this material via:
- Daily SRS review in JAVUMBO (SM-2 algorithm, 108-card Verbal Tenses deck).
- The Mid-E01 Prep Exam (formative checkpoint).
- Any classroom reinforcement during E01/E02.

### 1.3 Delivery

Google Forms, programmatically generated by `exam_prep/exam_01/scripts/create_exam_01_final_form.py`, following the same API pattern as `exam_prep/scripts/create_prep_exam_form.py`.

## 3. Design Principles

### 3.1 Structural Continuity

The exam mirrors the placement exam and prep exam in format to keep assessment instruments consistent across the semester.

| Dimension | Prep Exam | Final Exam (this document) |
|-----------|-----------|---------------------------|
| **Type** | Formative (optional) | Summative (graded) |
| **Tier 1 questions** | 10 | 10 |
| **Tier 2 questions** | 20 (T1 + 10 extra) | 20 (T1 + 10 extra) |
| **Tier 3 questions** | N/A | 30 (T1 + T2-extra + 10 more) |
| **Format** | Multiple choice, 4 options | Same |
| **Instructions** | Bilingual PT/EN | Same |
| **Delivery** | Google Forms | Same |
| **Grading** | Quiz mode, automatic | Same |

### 3.2 Content Freshness

All 30 questions in the new bank must be **original sentences** not appearing in `prep_exam_bank.json`. This prevents students who remember prep-exam questions from having a mechanical advantage.

The structural coverage (which tenses are tested at each tier) should match the prep exam's coverage table so that the exams remain comparable.

### 3.3 Tier 3 Question Design

Tier 3 students have demonstrated consistent high performance. Their 10 additional questions (Q21–Q30) should test:

- Conditional sentences (Type 1 and Type 2) with *will* / *would*
- Modal verbs in context (*must*, *should*, *might*)
- Simple passive voice (Simple Present and Simple Past)
- Reported speech (basic: He said that…)
- Time clauses with future reference (*when*, *as soon as* + Simple Present)

Use the same JSON schema as `exam_prep/bases/prep_exam_bank.json`:

```json
{
  "metadata": {
    "exam": "Exam 01 Final",
    "subject": "Verbal Tenses",

  },
  "questions": [
    {
      "id": "E01F_Q01",
      "order": 1,
      "tier": 1,
      "point_value": 1,
      "instruction_pt": "Escolha a opção correta para completar a frase.",
      "question_text": "She _____ to school every day.",
      "options": [
        {"text": "walk",   "is_correct": false},
        {"text": "walks",  "is_correct": true},
        {"text": "walked", "is_correct": false},
        {"text": "walking","is_correct": false}
      ]
    }
  ]
}
```

### 5.2 Validation Rules (enforced by the script)

| Rule | Value |
|------|-------|
| Total questions | 30 (or 20 if no Tier 3 students) |
| Tier 1 questions | exactly 10 (`"tier": 1`) |
| Tier 2-extra questions | exactly 10 (`"tier": 2`) |
| Tier 3-extra questions | exactly 10 (`"tier": 3`) — may be 0 if Tier 3 not present |
| Each question options | exactly 4 |
| Exactly one correct per question | enforced |
| `id` prefix | `E01F_Q` to distinguish from prep exam IDs (`PE_Q`) |

## 6. Google Form Structure

```
Section 0: Bilingual instructions + name selector
           → Students with assigned tier  → go directly to their tier section
           → Students without tier (blank) → go to Section X (Self-Selection)

Section X: Self-Selection (for unassigned students only)
           "Após os primeiros exercícios da disciplina, você se sente mais
            habilitado(a) para exames Tier 1, Tier 2 ou Tier 3?"
           → Tier 1 → Section 1
           → Tier 2 → Section 2
           → Tier 3 → Section 3

Section 1: Tier 1 questions (Q1–10)
           → SUBMIT at end

Section 2: Tier 2 questions (Q1–20, all 20)
           → SUBMIT at end

Section 3: Tier 3 questions (Q1–30, all 30)   ← only created if Tier 3 students exist
           → SUBMIT at end
```

> **Section ordering note**: The Forms API requires sections to appear in a fixed
> physical order, but routing can jump to any section by ID. Section X is created
> between Section 0 and Section 1 in the physical order; students with assigned tiers
> jump over it via `goToSectionId`, so they never see it.

### 6.1 Self-Selection Section (Section X)

This section is always created when there is at least one student with a blank tier.
It contains a single ungraded RADIO question with `goToSectionId` wired to each tier section:

```
"Após os primeiros exercícios da disciplina, você se sente mais habilitado(a)
 para exames Tier 1, Tier 2 ou Tier 3?
 (After the first exercises, which exam level do you feel ready for?)"

  ○ Tier 1 — básico / basic           → Section 1
  ○ Tier 2 — intermediário / intermediate → Section 2
  ○ Tier 3 — avançado / advanced      → Section 3
```

If there are no unassigned students, Section X is omitted entirely and the form
structure collapses to Section 0 → Section 1 / 2 / 3.

### 6.3 Form Settings

| Setting | Value |
|---------|-------|
| Quiz mode | Enabled |
| Collect email addresses | Yes |
| Limit to 1 response | Yes |
| Accept responses | Toggle ON manually after API creation |

## 7. Script Architecture

File: `exam_prep/exam_01/scripts/create_exam_01_final_form.py`

Based on `exam_prep/scripts/create_prep_exam_form.py`. Key differences:

### 7.1 Question Bank Loading

```python
t1 = [q for q in bank['questions'] if q['tier'] == 1]
t2 = [q for q in bank['questions'] if q['tier'] == 2]
t3 = [q for q in bank['questions'] if q['tier'] == 3]

if len(t1) != 10:
    raise ValueError(f"Expected 10 Tier 1 questions, found {len(t1)}")
if len(t2) != 10:
    raise ValueError(f"Expected 10 Tier 2-extra questions, found {len(t2)}")
if has_tier3 and len(t3) != 10:
    raise ValueError(f"Expected 10 Tier 3-extra questions, found {len(t3)}")
```

### 7.2 Roster Parsing

The v2 CSV uses different column names than the prep exam roster:

| Old roster (`curated_student_roster.csv`) | New roster (`curated_student_roster_v2.csv`) |
|------------------------------------------|---------------------------------------------|
| `Name` | `Nome` |
| `Suggested Tier` | `Suggested Tier` |
| — | `Curso` |
| — | `ID` |
| — | `Caminho` |

Tier mapping logic:

```python
raw = row['Suggested Tier'].strip()
if 'Tier 2' in raw:
    tier = 'tier2'
elif 'Tier 3' in raw:
    tier = 'tier3'
elif 'Tier 1' in raw:   # covers "Tier 1" and "Tier 1 🏁"
    tier = 'tier1'
else:
    print(f"  ℹ {row['Nome'].strip()} — no tier assigned, will route to self-selection section")
    continue
```

### 7.3 Form Sections

`build_form_structure` creates sections in physical order:

- **Section 0**: Instructions + name selector
- **Section X (Self-Selection)**: single routing question *(created only if unassigned students exist)*
- **Section 1 (Tier 1)**: 10 questions → Submit
- **Section 2 (Tier 2)**: 20 questions → Submit
- **Section 3 (Tier 3)**: 30 questions → Submit *(created only if Tier 3 students exist)*

### 7.4 Branching Logic

`setup_branching` is called after the first pass and reads back live item IDs. It patches:

1. **Name selector** (Section 0) — per-student `goToSectionId`:
   - Assigned Tier 1 → Section 1 ID
   - Assigned Tier 2 (incl. 🏁) → Section 2 ID
   - Assigned Tier 3 → Section 3 ID
   - No tier (blank) → Section X ID

2. **Self-selection question** (Section X) — per-option `goToSectionId`:
   - "Tier 1" option → Section 1 ID
   - "Tier 2" option → Section 2 ID
   - "Tier 3" option → Section 3 ID

```python
# Tier routing map built during setup_branching:
tier_to_section = {
    'tier1':    section_ids['tier1'],
    'tier2':    section_ids['tier2'],
    'tier3':    section_ids.get('tier3'),      # None if no Tier 3 section
    'unassigned': section_ids.get('self_select'),  # None if no Section X
}
```

## 8. Roster Parsing Notes

### 8.1 Encoding

The v2 roster has a BOM (`\ufeff`) at the start. Use `encoding='utf-8-sig'` when opening with `csv.DictReader` (unlike the v1 roster which required `latin-1`).

```python
with open(roster_path, 'r', encoding='utf-8-sig') as f:
    reader = csv.DictReader(f)
```

### 8.2 Students Without a Tier

~9 students have a blank `Suggested Tier`. The script routes them to **Section X (Self-Selection)** where they answer the self-assessment question and choose their own tier. No manual intervention is required — the form handles them automatically.

The script prints an info line for each unassigned student and creates Section X only when at least one is present.

### 8.3 Flagged Students (🏁)

Five Tier 1 students carry the `🏁` flag (`Tier 1 🏁`). They are routed to **Section 2 (Tier 2)**, consistent with the prep exam resolution strategy. The `🏁` character is safely handled by the string-contains check (`'Tier 1' in raw`).

5. **Obtain Google API credentials** — required before running the script.
   The script expects `credentials.json` at `placement_exam/credentials.json`.
   If the file already exists from the prep exam, skip to step 6.

   **To generate credentials from scratch:**

   a. Go to [https://console.cloud.google.com/](https://console.cloud.google.com/) and sign in with the Google account that owns the target Forms.

   b. Select (or create) a project. In the left menu go to **APIs & Services → Library**. Search for **Google Forms API** and click **Enable**.

   c. Go to **APIs & Services → OAuth consent screen**.
      - User Type: **External** → Create.
      - Fill in App name (e.g. "StudyAmigo Forms"), user support email, developer email.
      - Scopes: click **Add or Remove Scopes** → add `.../auth/forms.body` → Save and Continue.
      - Test users: add your own Google account email → Save and Continue → Back to Dashboard.

   d. Go to **APIs & Services → Credentials** → **+ Create Credentials → OAuth client ID**.
      - Application type: **Desktop app**.
      - Name: anything (e.g. "StudyAmigo CLI").
      - Click **Create** → **Download JSON**.
      - Rename the downloaded file to `credentials.json` and place it at:
        ```
        placement_exam/credentials.json
        ```

   e. The first time the script runs it will open a browser window asking you to
      authorize the app with your Google account. After authorization a
      `token.json` is saved to `placement_exam/token.json` and reused on
      subsequent runs.

6. **Generate the form** — run the script from its directory:
