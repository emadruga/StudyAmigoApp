## 1. Ponto de partida: o que E02 nos disse

### Dados de E02 (14/04/2026)

| Métrica | E01 | E02 |
|---------|----:|----:|
| Alunos no roster | 54 | 64 |
| Ativos | 44 | 44 |
| Taxa de participação | 81% | 69% |
| Revisões médias | 154 | 54 |
| Nota média (ativos) | — | 62.1 |
| Maturidade média | ~0% | 5.1% |
| Menções B ou acima | 3 | 4 |

### O problema central: consistência

50% dos alunos ativos estudaram apenas 1 ou 2 dias em E02. A queda de 81% → 69% de participação sobre um roster maior reforça que o engajamento é o gargalo principal — não a dificuldade do conteúdo.

### O que funcionou

- SM-2 corrigido: 10 alunos com cartões maduros (ivl ≥ 21d); Laís (74.2%) e Laryssa (66.7%) como destaque
- 91% dos alunos ativos criaram pelo menos um cartão (40/44)
- Maturidade média subiu de ~0% → 5.1% — tendência correta
- Mapeamento de contas secundárias via `account_map.csv` funcionou bem (14 mapeamentos)

### O que não funcionou

- 43% dos alunos ativos ficaram com F (nota < 60)
- 4 alunos não criaram nenhum cartão (apenas revisaram o baralho compartilhado)
- Rogério Gabriel: 141 revisões, 0 cartões criados, 38.6% retenção — padrão sem estrutura de aprendizado
- Flags de comportamento suspeito: RET100 com maturidade 0% (Adriany, Matheus, Ana Luiza)

### Tier 3 — Expanding

**Texto**: *"From Chess Prodigy to Nobel Laureate: The Mind Behind AlphaFold"* — **mesmo texto de E02**
**Tamanho**: 490 palavras
**Justificativa**: o texto tem vocabulário editorial denso o suficiente para uma segunda passagem produtiva. Em E02 a maioria dos alunos criou 10–15 cartões; uma segunda passagem com critério mais fino deve revelar 10+ palavras novas.
**Seleção de palavras**: autônoma — o aluno aplica a auditoria produtiva (usa em frase nova? colocação? classe gramatical?) sem lista fornecida pelo professor.
**Formato de cartão**: definição em inglês — verso com `palavra (classe) = definição EN` + `☞ colocações` + tradução da frase

## 7. Critérios de avaliação

Fórmula mantida de E02:

```
Nota = 0.25×V + 0.25×C + 0.30×Q + 0.20×E
```

| Componente | Variáveis |
|------------|-----------|
| **V — Volume** | revisões totais / cards criados |
| **C — Consistência** | dias de estudo / distribuição temporal |
| **Q — Qualidade** | retenção (70%) + maturidade (30%) |
| **E — Engajamento** | tempo de resposta + distribuição dos botões Again/Hard/Good/Easy |

> **Maturidade** (ivl ≥ 21d): com 26 dias de E03, alunos que criarem cartões na primeira semana e revisarem consistentemente poderão ter cartões maduros ao final.

## 8. Avaliação programática de aderência ao método

Em E03 a adequação ao método é avaliada automaticamente pelo script `assess_card_quality.py`. O script é **diagnóstico separado** — não altera a fórmula de nota do `grade_exercise_v2.py`.

### Script

```
placement_exam/planning_E03/scripts/assess_card_quality.py
```

### Heurísticas por cartão (score 0–6 → normalizado 0–100%)

Cada cartão é avaliado em 3 critérios independentes (0–2 pts cada):

| Critério | Pontuação | Lógica |
|----------|-----------|--------|
| **Formato da frente** | 0–2 | 2: frase ≥ 4 palavras + `[MAIÚSCULO]` · 1: frase sem marcação · 0: palavra isolada |
| **Formato do verso** | 0–2 | Regras diferentes por tier (ver abaixo) |
| **Evidência de processo** | 0–2 | 2: similaridade < 0.60 com texto-fonte · 1: 0.60–0.85 · 0: ≥ 0.85 (cópia) |

**Verso por tier:**

| Tier | 2 pts | 1 pt | 0 pts |
|------|-------|------|-------|
| T1 | padrão `palavra = tradução` | texto sem `=` | vazio |
| T2 | classe gramatical + `☞` | apenas um dos dois | nenhum |
| T3 | `☞` + definição EN (sem `= tradução_PT`) | `☞` mas com tradução PT | sem `☞` |

### Outputs

| Arquivo | Conteúdo |
|---------|----------|
| `E03_card_quality_detail.csv` | Uma linha por cartão: `student_id, name, tier, card_id, front, back, score_front, score_back, score_process, total_score, pct_score` |
| `E03_card_quality_summary.csv` | Uma linha por aluno: médias dos 3 scores + `low_quality_cards` + `flag_mostly_copied` |

### Flags de alerta no resumo

| Flag | Condição |
|------|----------|
| `low_quality_cards` | Número de cartões com `pct_score < 33%` |
| `COPIA` | Mais de 50% dos cartões com `score_process = 0` |

### Comando

```bash
placement_exam/.venv/bin/python \
    placement_exam/planning_E03/scripts/assess_card_quality.py \
    --start 2026-04-21 --end 2026-05-17 \
    --roster exam_prep/exam_01/bases/curated_student_roster_v2.csv \
    --account-map placement_exam/planning_E02/account_map.csv \
    --admin-db ~/.cache/studyamigo/SNAPSHOT/admin.db \
    --user-db-dir ~/.cache/studyamigo/SNAPSHOT/user_dbs \
    --detail-output placement_exam/planning_E03/E03_card_quality_detail.csv \
    --summary-output placement_exam/planning_E03/E03_card_quality_summary.csv
```

> Use `--deck-prefix PASSAGE_E02` para testar o script contra um snapshot de E02.
